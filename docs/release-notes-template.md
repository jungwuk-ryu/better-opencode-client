# 릴리즈 노트 템플릿

작성일: 2026-03-31  
원칙: 기술 변경보다 `사용자에게 생긴 변화`를 먼저 적는다.

## 1. 템플릿

```md
# vX.Y.Z

릴리즈 날짜: YYYY-MM-DD

## 한 줄 요약

이번 릴리즈는 어떤 사용자 경험을 개선했는가.

## 새로 좋아진 점

- 무엇이 더 쉬워졌는지
- 어떤 작업이 더 빨라졌는지
- 어떤 오류가 덜 헷갈리게 바뀌었는지

## 안정성 변화

- 어떤 실패가 덜 자주 보이는지
- 어떤 복구 경로가 좋아졌는지
- 어떤 플랫폼 제약이 있는지

## 주요 화면 / 흐름

- 첫 연결
- 작업 재진입
- triage
- git 루프

## 사용자 주의 사항

- 이번 버전에서 아직 지원하지 않는 것
- 환경별 차이
- 필요한 권한

## 검증

- 확인한 테스트
- 수동 검증한 흐름

## 다음 계획

- 다음 릴리즈에서 다룰 큰 항목
```

## 2. 작성 규칙

- 첫 문장은 기술보다 사용자 가치로 시작한다.
- 버그 수정도 사용자가 체감하는 언어로 쓴다.
- 불완전한 항목은 숨기지 말고 제한으로 적는다.
- `experimental` 성격의 기능은 별도 문단으로 분리한다.
