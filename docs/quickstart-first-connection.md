# 첫 연결 빠른 시작

작성일: 2026-03-31  
대상: `better-opencode-client (BOC)`의 첫 설치 사용자

이 문서는 설치 직후 `첫 연결`까지 가는 최소 경로만 다룬다.  
현재 앱은 서버 프로필 저장, probe 기반 상태 확인, 연결 프로필 import, deep link 진입을 지원한다.

## 1. 목표

- 앱을 설치한 뒤 빠르게 서버를 등록한다.
- 등록 직후 서버 상태를 확인한다.
- 연결 실패 시 원인을 대략 `network / TLS / auth / capability` 수준으로 읽는다.
- 이후에는 같은 프로필을 다시 열어 바로 워크스페이스로 진입한다.

## 2. 준비 사항

- Flutter 개발 환경
- 접속할 OpenCode 서버의 주소
- 필요한 경우 사용자명과 비밀번호
- 공유된 연결 링크 또는 import payload

## 3. 설치

1. 저장소 의존성을 받는다.

   ```bash
   flutter pub get
   ```

2. 앱을 실행한다.

   ```bash
   flutter run
   ```

3. 처음 열면 홈 화면에서 서버 목록과 현재 상태를 확인한다.

## 4. 첫 연결 방법

### 방법 A. 수동 등록

1. 홈 화면에서 서버 추가 화면으로 들어간다.
2. 서버 이름과 base URL을 입력한다.
3. 인증 정보가 필요하면 사용자명과 비밀번호를 넣는다.
4. 저장 후 probe 결과를 확인한다.
5. 연결 상태가 정상으로 뜨면 해당 서버를 선택한다.

### 방법 B. 공유 링크 import

1. 공유받은 연결 링크를 연다.
2. 앱이 `Import Connection` 시트를 열면 내용을 확인한다.
3. `Auth`, `Expires`, `Duplicate` 정보를 검토한다.
4. 문제가 없으면 저장한다.
5. 이미 같은 서버가 있으면 기존 프로필을 업데이트한다.

### 방법 C. deep link 진입

1. 외부에서 `opencode-remote://connect?payload=...` 형태의 링크를 연다.
2. 앱이 연결 import 화면으로 이동한다.
3. 검증이 통과하면 저장한다.

## 5. 성공 확인

- 홈 화면에서 서버 상태가 보인다.
- probe가 최신 상태로 갱신된다.
- 워크스페이스로 들어가면 마지막 세션과 프로젝트 컨텍스트가 이어진다.
- 같은 서버를 다시 열면 저장된 프로필을 재사용한다.

## 6. 실패할 때 보는 순서

1. 서버 주소가 정확한지 확인한다.
2. 인증 정보가 맞는지 확인한다.
3. TLS 오류인지, 네트워크 차단인지 확인한다.
4. capability가 부족한 서버인지 확인한다.
5. 문제가 계속되면 probe 상태를 기준으로 다시 저장한다.

## 7. 현재 지원 범위

- 서버 프로필 저장
- probe 기반 상태 표시
- 연결 import 검증
- deep link 진입
- 같은 서버 프로필 업데이트

## 8. 제한 사항

- 이 문서는 첫 연결 경로만 다룬다.
- 고급 진단 마법사나 안내형 복구 플로우는 아직 공개 문서 수준에서만 정리했다.
- universal link, web continuity, PWA 중심 흐름은 현재 주 경로가 아니다.
