# 외부 문서 검증 체크리스트

작성일: 2026-03-31  
목적: 깨끗한 환경에서 문서가 실제로 설치와 첫 연결을 안내하는지 확인한다.

## 1. 검증 환경

- 새 Flutter 설치 또는 깨끗한 개발 환경
- 이전 앱 데이터가 없는 상태
- 테스트 서버 주소 또는 샘플 연결 링크
- iOS 또는 Android 한 개 이상

## 2. 체크리스트

| 항목 | 절차 | 기대 결과 | 결과 |
| --- | --- | --- | --- |
| 설치 안내 | README 또는 quickstart만 보고 시작 | `flutter pub get`와 `flutter run`이 바로 보인다 |  |
| 첫 연결 | 서버를 수동 등록한다 | probe 결과가 표시된다 |  |
| import | 공유 링크를 연다 | import 시트가 뜨고 검증 상태가 보인다 |  |
| deep link | custom scheme 링크를 연다 | 앱이 연결 import 화면으로 이동한다 |  |
| workspace 진입 | 저장된 서버를 다시 연다 | 이전 workspace 문맥이 이어진다 |  |
| triage | inbox를 연다 | 질문/승인/이벤트가 구분된다 |  |
| git 루프 | git sheet에서 commit 또는 push를 시도한다 | 상태가 갱신된다 |  |
| fallback | 실패 상황을 만든다 | terminal fallback 또는 오류 메시지가 명확하다 |  |
| mobile ergonomics | 한 손으로 주요 CTA를 눌러 본다 | 핵심 액션에 손이 닿는다 |  |
| 문서 문장 | README/quickstart 문구를 읽는다 | 과장 없이 현재 기능과 맞는다 |  |

## 3. 합격 기준

- 첫 연결까지의 경로가 5분 안에 설명된다.
- 연결 실패 원인이 문서와 화면에서 겹친다.
- 문서가 구현되지 않은 기능을 이미 있는 것처럼 말하지 않는다.
- 지원 경로와 다음 행동이 분명하다.
