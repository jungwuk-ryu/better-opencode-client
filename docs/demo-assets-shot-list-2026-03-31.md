# 데모 자산 샷 리스트

작성일: 2026-03-31  
목적: 외부 공유용 데모 자산을 어떤 순서와 메시지로 만들지 정리한다.

## 1. 사용 원칙

- 실제 캡처가 가능하면 실화면을 쓴다.
- 아직 캡처하지 못했다면 `스토리보드`로 명시한다.
- 없는 기능을 데모에 넣지 않는다.

## 2. 샷 목록

| 순서 | 장면 | 메시지 | 자산 형태 | 비고 |
| --- | --- | --- | --- | --- |
| 1 | README 첫 화면 | 이 제품이 무엇인지 한 문장으로 전달 | 정적 스크린샷 | 제품 소개 시작점 |
| 2 | 첫 연결 | 서버를 바로 등록하고 상태를 본다 | 스크린샷 또는 짧은 GIF | quickstart 핵심 |
| 3 | import 시트 | 공유 링크를 검토하고 저장한다 | 스크린샷 | deep link 흐름 |
| 4 | workspace 복귀 | 마지막 문맥이 이어진다 | GIF | 재진입 가치 강조 |
| 5 | inbox | 질문과 승인을 한 곳에서 처리한다 | GIF | mobile triage |
| 6 | project actions | 반복 작업을 한 번에 연다 | GIF | 운영성 강조 |
| 7 | git sheet | stage, commit, push의 최소 루프 | GIF | completion loop |
| 8 | voice input | 말해서 prompt를 입력한다 | 짧은 GIF | mobile-native 차별화 |
| 9 | quick attach | 이미지 또는 파일을 prompt에 붙인다 | 스크린샷 | 입력 비용 절감 |
| 10 | support / docs | 지원 경로와 문서가 보인다 | 정적 스크린샷 | 신뢰 신호 |

## 3. 캡션 초안

- `첫 연결까지의 경로를 짧게 만들었다.`
- `공유 링크로 서버를 다시 열 수 있다.`
- `질문, 승인, 작업을 한 화면에서 정리한다.`
- `작업을 끝내기 위한 git 최소 루프를 앱 안에 넣었다.`

## 4. 제작 메모

- GIF는 5초에서 12초 사이로 짧게 자른다.
- 배경 설명은 한 줄만 둔다.
- 핵심은 멋진 장면보다 흐름의 단순함이다.
